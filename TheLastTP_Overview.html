TheLastTP

by: Nolan Flodine

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I was inspired by both the game Space Invaders and the current Corona Virus situation we are in right now.

2. What I Learned

I learned from user testing that the game could be very difficult in the first few tries, but the testers that played the game several times were able to make it to the end and win the game.

3. Architecture of My Game

The game starts with a start page that was designed on gimp and tells the player to either press space for instructions or enter to start. Space would lead them to another page with the instructions that was also designed in gimp. If they clicked enter than the game will start and infected people would come up from the bottom of the screen and the player has to shoot the the infected people with sanitizer to disinfect them. The player starts with 20 shots and can hit refill power ups that float across the screen. There are also Lysol spray power ups that cause rain to wash the infected away and snail power ups that slow the rate of which the infected spawn. The infected rate of spawn increases over time and if the player disinfects 100 people without dying he wins the last TP and a graphic is shown that was made in gimp. If the player gets infected than the player will start to shake and he will get infected. I also included sound files for when the player hits the power ups, wins, dies, or is in the start menu. There is also background music that goes on throughout the game. The classes were the power up class, drop class, infected class, and Lysol class. They all create arrays and have several methods. The drop and Lysol class also contain Collision methods.

4. Technical Implementation

A difficult technical implementation were the power ups. They spawn at a random y value and random side of the screen. If the Lysol power up is hit than very realistic rain spawns. In the rain there are different stroke lengths to act as is some rain drops are closer than others. The closer rain drops fall faster and the further rain drops fall slower. There is also gravity so as they go on they get faster. When a rain drop touches the infected than the infected gets set to an x and y position far off the map because Processing does not allow for a particular image in an array to be deleted. Also I created a timer variable using millis() and I was able to make the rain fall for 5 seconds using this time variable. Finally, a spray noise plays for the 5 second and stops when the rain stops.

5. Most Challenging Part

The most challenging part of the project was the managing of images in arrays which I received help for one array and was able to make the other arrays by modifying the code from the first array of images. I also had problems changing the game state so I created a separate method that handles game state.

6. Future Fixes and Updates

I would improve a bug I had that the power ups could sometimes spawn at the same time as the refill and they could sometimes change direction in the middle of the screen. This happened pretty rarely and did not effect the game much, but it would still be good to find a solution.
